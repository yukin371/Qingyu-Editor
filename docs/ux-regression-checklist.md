# Qingyu-Editor 体验冻结期回归清单

> 适用阶段：基本功能冻结后，进入用户体验优化与稳定性回归。除阻断 bug 外，不新增主流程功能。

## 0. 回归原则

- 每轮只处理一组体验问题，避免把主流程、工具页、AI、样式和数据 owner 混在同一提交。
- 优先验证真实作者路径：创建项目、写章节、管理设定、AI 辅助、保存恢复。
- 工具入口只保留一个清晰 owner；发现重复入口时优先收敛到设置页、右栏或 overlay 的既有 owner。
- 明暗主题都要检查，不允许新增硬编码白底、黑字或脱离 theme token 的面板。
- 长篇场景必须验证窗口化定位，不允许全量铺开几百/几千章、事件或分支节点。

## 1. 主流程 Smoke

- 启动编辑器，进入作者工作台。
- 新建项目后自动生成基础骨架：第一卷、第一章。
- 新建项目后自动跳转到第一章，标题行可直接编辑。
- 新建章节按当前卷栈式追加，第二卷内添加章节不串到其他卷。
- 切换章节后正文内容、标题、字数、保存状态保持一致。
- 自动保存有防抖，频繁输入不会卡顿；停止输入后保存状态更新。
- 字数统计只统计汉字、单词；标点不计入，连续字母拟声词按 1 词计。

## 2. 布局与入口

- 左侧边栏展开/隐藏按钮可点击，快捷键与按钮行为一致。
- 右侧工具栏 AI / 设定 / 审查 / 校对 / 灵感可打开并保持当前工具状态。
- 底部场景舞台可拖拽调整高度，不遮挡正文，不随切章自动换拍。
- 顶部设置入口是工作区设置唯一入口；快捷键只在设置页内配置。
- 更多操作菜单只保留分享、场景舞台、布局预设等轻动作，不再挂重复设置页。
- Overlay 工具只通过结构舞台、资产总览、关系图谱、时间线、分支入口打开，不回流主编辑区。

## 3. 设定与资产

- 正文输入 `@名称` 后可创建资产，并把引用写回正文。
- `@资产` 自动扫描后能出现在右侧设定的本章/本卷投影中。
- 资产总览可新建、编辑、删除全局资产。
- 右侧设定分类右侧 `+` 可快速新建对应类型全局资产。
- 删除正文里的 `@资产` 只解除章节引用，不删除全局资产。
- 未建档 `@名称` 保持待确认，不默认强判为角色。
- 空资产分类、无搜索结果、无详情选择时显示轻量 empty，不出现大段说明。

## 4. AI 主链路

- Provider 设置页可切换系统服务 / 用户 API。
- 用户 API 支持服务地址、模型、API Key 配置，密钥显示脱敏状态。
- 健康检查能显示当前模式、配置完整性和失败原因。
- 发送“改写当前章节”后只出现正文 inline diff，不静默覆盖正文。
- 发送跨章节请求时，多命中先显示候选卡，不自动切章。
- AI evidence 能说明本次参考了章节、场景、资产或结构摘要。
- 无正文、无 provider、请求失败、超时/取消都有可理解反馈。

## 5. 基本工具

- 结构舞台默认展示当前章节、附近章节、当前节拍、大纲绑定和设定入口。
- 结构舞台在多卷项目中不乱序，不因点击章节改变原始位置。
- 场景舞台覆盖范围只能从当前章节向前连续选择，不出现断档范围。
- 大纲和节拍联动清晰：大纲是规划草稿，节拍是当前剧情段承诺。
- 灵感工具停留在项目内，不跳转模板中心。

## 6. 高级工具

- 时间线默认按事件窗口展示，空态不突兀。
- 时间线定位命中后跳到对应事件区段，事件顺序不乱。
- 关系图谱自动消费章节/卷/全局资产引用投影。
- 分支工具默认不把普通小说大纲伪装成复杂分支；无分支时可看示例。
- 大量分支路线默认收束，定位后保持路线原位置关系。

## 7. 视觉与暗色模式

- 工作区、右栏、底栏、overlay、设置页在暗色模式下无强烈白底。
- 按钮、输入框、empty、分割线、选中态使用统一 theme token。
- 同一层级不重复标题；已有主标题时不再补副标题和说明段。
- 卡片嵌套不超过必要层级，优先用分栏、分割线和密度区分信息。
- 工具 empty 只说明当前状态和一个下一步动作，不写教程式长文案。

## 8. 长篇压力样本

- 500+ 章节项目：左侧目录、结构舞台、章节定位不卡顿。
- 2000+ 章节项目：搜索章节后不把命中项移到首行，不改变原始顺序。
- 500+ 时间线事件：只渲染当前窗口，定位跳到命中窗口。
- 100+ 分支路线：路线列表分组收束，节点检视仍可操作。
- 大量资产引用：右侧设定和资产总览仍按分类/作用域过滤，不全量堆叠。

## 9. 验证命令

```bash
npm run type-check
npm run test -- WorkspaceTopbar WorkspaceSettingsPanel ShortcutSettingsPanel
npm run test -- EncyclopediaView AssetListPanel TimelineOutlineView StoryBranchView
npm run test -- ProjectWorkspace WorkspaceSceneStagePanel
npm run test:e2e:core -- tests/e2e/writer-workflow.spec.ts
git diff --check
```

## 10. 浏览器手工回归

- 打开编辑器并创建一个新项目。
- 写入第一章正文，插入 `@角色`，确认右栏设定自动出现。
- 新建第二章，确认场景舞台不自动换拍。
- 打开设置页，切到快捷键页签，确认更多操作中没有重复快捷键入口。
- 切换暗色模式，检查主编辑区、右栏、底栏、资产总览、时间线、分支空态。
- 发送一次 AI 改写请求，确认正文出现 inline diff。
