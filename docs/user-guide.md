# Qingyu-Editor 产品说明书

> 适用版本：v0.1.0-beta。工作台内可通过“更多操作 -> 使用文档”打开同款内嵌说明书。

## 1. 产品定位

Qingyu-Editor 是面向长篇小说作者的独立写作工作台。它把“创建作品骨架、写当前章节、沉淀设定、AI 回审、复杂工具辅助”收进同一个项目工作区，让作者大多数时间只围绕当前章节和当前场景推进。

核心原则：

- 作者始终拥有最终决定权，AI 只辅助生成、审查、整理和建议。
- 正文修改必须可审阅，AI 不静默覆盖正文。
- 基础工具保持低学习成本，高级工具只在复杂作品需要时使用。
- 长篇项目必须支持定位、窗口化和分层管理，避免几千章场景下全量堆叠。

## 2. 从灵感到第一章

第一次使用时，只要先完成项目骨架，再进入第一章写作。系统不会要求作者先学完整套工具。

1. 在工作台点击“新建项目”，填写作品名后创建。
2. 系统自动生成第一卷、第一章，并进入章节标题行。
3. 正文区保持空白，作者先命名章节，再开始写正文。
4. 如果还没有完整设定，可以先写一句灵感，再逐步让 AI 协助扩展。

截图演示：新项目骨架

```text
工作台 -> 新建项目
┌ 左侧目录 ─────────┬ 标题行 ─────────────┬ 正文区 ────────────┐
│ 第一卷 / 第一章   │ 第一章，可直接改名   │ 空白，等待落稿      │
└───────────────────┴────────────────────┴────────────────────┘
```

## 3. 日常写作主界面

工作区只围绕三件事展开：左侧管理章节，中间写正文，右侧和下侧提供当前写作需要的辅助。

1. 左侧栏管理卷和章节，新建章节会追加到当前卷末尾。
2. 中间标题行只负责章节名，正文区负责正文、图片、AI diff。
3. 右侧栏保留 AI、设定、灵感、审查、校对几个常用入口。
4. 下侧场景舞台管理当前剧情段，不会因为切章自动换拍。

截图演示：工作区分区

```text
┌ 左栏：卷/章节 ┬ 主编辑区：标题 + 正文 + diff ┬ 右栏：AI/设定/审查 ┐
├──────────────┴──────────────────────────────┴──────────────────┤
│ 下侧：当前场景与节拍                                             │
└──────────────────────────────────────────────────────────────────┘
```

## 4. 设定、资产与 @ 引用

手动创建永远是全局资产；本章和本卷只是系统从正文中自动检测出的局部投影。

1. 正文输入 `@名称`，可创建或引用角色、地点、物件、组织、概念。
2. 右侧设定按“本章 / 本卷 / 全局”查看，从小范围到全局逐层理解。
3. 分类右侧的 `+` 用于快速创建对应类型的全局资产。
4. 删除正文里的 `@资产` 只解除当前章节引用，不删除全局资产。

截图演示：右侧设定

```text
右侧设定
┌ 范围切换：本章 / 本卷 / 全局 ┐
├ 分类树：角色 + 地点 + 物件   │
└ 详情区：摘要 / 引用 / 编辑   ┘
```

## 5. AI 辅助写作

AI 不是替作者接管作品，而是帮助写、审、整理。所有正文改动都必须进入可审阅 diff。

1. 设置页可选择系统服务，或配置自己的 OpenAI 兼容 provider。
2. 用户 API 支持多个 provider 配置槽，导出配置不会包含明文 Key。
3. 续写、扩写、改写会挂 inline diff；作者确认后才进入正文。
4. 总结、审校、整理只输出建议、候选卡或任务卡，不静默修改正文。

截图演示：AI 与 Provider

```text
AI 面板 / Provider 设置
┌ AI 模式：写作 / 整理 / 回审 / 问答 ┐
├ Provider：系统服务 / 用户 API       │
└ 正文结果：inline diff 后确认        ┘
```

## 6. 大纲、结构舞台与当前场景

大纲是规划草稿，当前场景是当前剧情段承诺，结构舞台是把章节、大纲、节拍和设定聚合起来看的入口。

1. 用大纲记录卷、章节目标和草稿想法，不要求一次写完整。
2. 用下侧当前场景写清目标、冲突、完成条件和下一拍。
3. 用结构舞台检查当前章节是否入纲、节拍是否明确、附近章节是否连贯。
4. 点击“进入下一拍”才推进节拍；切换章节不会自动推进。

截图演示：结构聚合

```text
结构舞台 / 当前场景
┌ 当前章节：第二章 ┬ 当前拍：旧友现身 ┐
├ 大纲绑定：卷一/起势段              │
└ 下一步：进入下一拍                 │
```

## 7. 高级工具何时使用

商业写作可以只用基础工具；关系图谱、时间线、故事分支用于复杂作品，不是每个作者都必须打开。

- 关系图谱：人物关系复杂、阵营变化明显时使用。
- 时间线：多线叙事、倒叙、长跨度事件时使用。
- 故事分支：互动叙事或多路线作品使用，普通小说可不使用。
- 长篇项目中，高级工具默认窗口化展示，不全量铺开几千个节点。

截图演示：高级工具 overlay

```text
Overlay 工具
┌ 关系图谱：角色 / 阵营 / 关系 ┐
├ 时间线：事件窗口 + 定位      │
└ 故事分支：路线 / 节点 / 结局 │
```

## 8. 安全边界与常见问题

- AI 不自动批量改多个章节；多章节请求默认返回计划或候选。
- 全局资产删除只能在明确资产管理动作中发生。
- Provider 配置导出不包含明文 API Key，浏览器环境只保存本次会话密钥。
- 如果某个工具看起来太复杂，先回到结构舞台、右侧设定和当前场景三个基础入口。
