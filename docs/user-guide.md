# Qingyu-Editor 用户使用指南

> 适用版本：v0.1.0-beta

## 1. 产品定位

Qingyu-Editor 是面向长篇小说作者的独立写作工作台。它优先服务“从灵感到章节落稿”的主流程，并把大纲、节拍、资产、时间线、关系图谱、AI 辅助等工具收敛到一个项目工作区中。

核心设计原则：

- 作者始终拥有最终决定权，AI 只辅助生成、审查、整理和建议。
- 正文修改必须可审阅，AI 不静默覆盖正文。
- 基础工具保持低学习成本，高级工具只在复杂作品需要时使用。
- 长篇项目必须支持定位、窗口化和分层管理，避免几千章场景下全量堆叠。

## 2. 推荐创作流程

### 2.1 从项目开始

1. 在工作台创建项目。
2. 新项目默认生成基础骨架：第一卷、第一章。
3. 创建完成后自动进入新章节，并聚焦章节标题。
4. 标题行只写章节名，正文区不重复渲染标题。

### 2.2 先搭骨架

- 使用项目初始化或模板能力整理作品定位、题材、目标读者、核心爽点、主题和风格。
- 使用大纲规划卷、篇章、章节目标。
- 结构舞台作为聚合页面，用来查看章节、节拍、大纲绑定、资产入口和 AI 整理入口。

### 2.3 再写当前场景

- 章节是保存和编辑的最小单位。
- 场景/节拍是叙事推进单位，可以覆盖多个连续章节。
- 下侧“当前场景”用于管理当前剧情段的目标、冲突、完成条件和下一拍。
- 切换章节不会自动进入下一拍；只有点击“进入下一拍”才推进。

### 2.4 写作中管理资产

- 在正文中输入 `@名称` 可以快速创建或引用角色、地点、物件、组织、概念。
- 手动创建的资产都是全局资产。
- 本章/本卷资产是系统从正文引用自动检出的局部投影。
- 删除正文中的 `@资产` 只解除章节引用，不删除全局资产。
- 全局资产的增删改查在右侧设定或资产总览中完成。

### 2.5 使用 AI

AI 的推荐使用方式：

- 写：扩写、续写、改写当前章节或选区，结果进入正文 inline diff。
- 审：检查节奏、设定一致性、人物动机、语言问题，只输出建议。
- 整理：从章节中整理资产、节拍、大纲差异或任务卡，不直接改正文。

AI 请求会尽量只携带必要上下文：当前章节、选区、当前场景、资产摘要、结构摘要和证据卡，不默认塞入全书全文。

## 3. 主界面说明

### 3.1 顶部栏

- 左侧显示项目入口。
- 中间显示当前章节、保存状态和当前工具。
- 右侧提供保存、导出、设置、快速工具和更多操作。
- 更多操作中包含分享、场景舞台、布局预设和使用文档。

### 3.2 左侧栏

- 管理卷和章节。
- 新建章节按当前卷栈式追加。
- 长篇项目中应通过搜索、定位当前、卷级范围减少滚动成本。

### 3.3 主编辑区

- 正文编辑、标题编辑、AI inline diff 接受/放弃都在这里完成。
- 保存和字数统计会防抖更新。
- 字数统计只统计汉字和单词；标点不计入，连续字母拟声词按一个词计。

### 3.4 右侧栏

- AI：聊天、写作、审查、整理的主入口。
- 设定：全局/本章/本卷资产快查、快建、快编。
- 审查：Story Harness、审查包和建议队列。
- 校对：基础文本问题检查。
- 灵感：项目内灵感、模板、黄金三章、核心承诺。

### 3.5 下侧栏

- 管理当前场景和当前拍。
- 默认使用横向三栏展示，减少上下滚动。
- 覆盖范围只能从当前章节向前选择连续章节。

### 3.6 Overlay 工具

- 结构舞台：基本工具聚合入口，适合默认打开。
- 资产总览：全局资产管理。
- 关系图谱：角色/组织/资产关系。
- 时间线：事件顺序、因果和时间跨度。
- 故事分支：视觉小说、互动小说、游戏叙事路线。

## 4. AI Provider 配置

进入“设置 -> AI Provider”：

- 系统服务：使用系统配置的远程 AI 服务。
- 用户 API：配置自己的 OpenAI 兼容 provider。

用户 API 支持：

- 常见 provider 模板：Qwen、DeepSeek、Kimi、GLM、Gemini、GPT、Claude、本地 Ollama、LM Studio、自定义。
- 多 provider 配置槽：可新增、切换、删除配置。
- 用途模型：写作、审查、整理可预留不同模型偏好。
- API Key 不回显：输入后保存，输入框立即清空，只显示密钥状态。
- 导出配置不包含明文 API Key。

桌面端通过系统 secret store 保存真实 API Key；浏览器测试环境只保存到当前会话。

## 5. 高级工具何时使用

- 结构舞台：日常全局管理，建议常用。
- 当前场景：日常写作节拍管理，建议常用。
- 资产：角色、地点、物件、组织、概念多时必用。
- 关系图谱：人物关系复杂、阵营变化明显时使用。
- 时间线：多线叙事、倒叙、长跨度事件时使用。
- 故事分支：互动叙事或多路线作品使用，普通小说可不使用。

## 6. 安全规则

- AI 不自动批量修改多个章节。
- AI 不自动创建章节落盘，除非用户明确确认计划。
- 全局资产删除只在资产管理动作中发生。
- 删除正文引用不会删除全局资产。
- Provider 配置导出不包含明文 Key。
